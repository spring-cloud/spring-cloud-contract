4. Spring Cloud Contract Verifier Setup

You can set up Spring Cloud Contract Verifier in the following ways:

  • As a Gradle project
  • As a Maven project
  • As a Docker project

4.1 Gradle Project

To learn how to set up the Gradle project for Spring Cloud Contract Verifier, read the following sections:

  • Section 4.1.1, “Prerequisites”
  • Chapter 5, Add Gradle Plugin with Dependencies
  • Section 5.1, “Gradle and Rest Assured 2.0”
  • Section 5.2, “Snapshot Versions for Gradle”
  • Section 5.3, “Add stubs”
  • Section 5.5, “Default Setup”
  • Section 5.6, “Configure Plugin”
  • Section 5.7, “Configuration Options”
  • Section 5.8, “Single Base Class for All Tests”

  • Section 5.10, “Invoking Generated Tests”
  • Section 5.11, “Pushing stubs to SCM”
  • Section 5.12, “Spring Cloud Contract Verifier on the Consumer Side”

4.1.1 Prerequisites

In order to use Spring Cloud Contract Verifier with WireMock, you muse use either a Gradle or a Maven plugin.

Warning

If you want to use Spock in your projects, you must add separately the spock-core and spock-spring modules. Check Spock docs for more information
